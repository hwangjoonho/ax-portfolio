검토 후 보류한 기술

Semantic Chunking
청킹 시점 임베딩 API 호출 비용 추가, 구조화 문서에서 효과 불확실 → 비구조화 문서 증가 시 재검토
Late Chunking (Jina)
임베딩 인프라 전면 교체 필요, 호환성 문제 → 차세대 버전에서 검토
Agentic Chunking
문서당 LLM 수백 회 호출, 인덱싱 비용 폭증 → ROI 개선 시 재검토
Parent-Child 구조
근본 원인(소형 청크)을 우회하는 방식 → v2.2로 근본 해결 완료
Reranker (Cross-encoder)
현재 Hybrid+CCH 조합으로 충분한 품질 확보 → 정밀도 추가 필요 시 도입
