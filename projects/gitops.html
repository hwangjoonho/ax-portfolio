Features
주요 기능
01
YAML 기반 선언적 설정
ci-config.yaml 하나로 배포 브랜치, 포트, 리소스, 알림 등 모든 설정 관리. Jenkinsfile 수정 없이 YAML만 변경.
02
Auto-MR + Auto-Merge
feature/fix/hotfix 브랜치 push 시 테스트 성공하면 자동 MR 생성 및 Merge. ci-config.yaml에서 on/off.
03
멀티모듈 지원
Gradle 멀티모듈에서 모듈별 독립 Docker 이미지 빌드 및 개별 K8s Deployment/Service 자동 생성.
04
테스트 게이트
JUnit 테스트 실패 시 빌드 즉시 중단, 배포 차단. 테스트 통과된 코드만 운영 환경에 반영.
05
Auto .dockerignore
프로젝트에 .dockerignore 없으면 빌드 시 자동 생성. 프론트/백엔드별 최적화된 기본값 적용.
06
Google Chat 알림
빌드 시작/성공/실패 시 Google Chat 웹훅으로 팀 알림 전송.

ci-config.yaml 
# 배포 대상 브랜치

    containerPort: 80
    servicePort: 9004

# Auto-MR

  enabled: true
  targetBranch: "dev"
